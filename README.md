১. DOM Selectors এর মধ্যে পার্থক্য:
getElementById: এটি সবচেয়ে দ্রুত কাজ করে। যখন আমরা নিশ্চিত থাকি যে আমাদের একটি নির্দিষ্ট ID-র এলিমেন্ট দরকার, তখন এটি ব্যবহার করি। এটি একটি মাত্র এলিমেন্ট রিটার্ন করে।

getElementsByClassName: এটি একই ক্লাসের যতগুলো এলিমেন্ট আছে সবগুলোকে একটি লিস্ট (HTMLCollection) আকারে দেয়।

querySelector & querySelectorAll: এগুলো সবচেয়ে আধুনিক। এখানে CSS এর মতো .class বা #id দিয়ে সার্চ করা যায়। querySelector দেয় শুধু প্রথম এলিমেন্টটি, আর querySelectorAll দেয় সব এলিমেন্ট একটি NodeList হিসেবে।

২. নতুন Element তৈরি ও যোগ করার প্রক্রিয়া:
DOM-এ নতুন এলিমেন্ট যোগ করার জন্য আমরা মূলত তিনটি ধাপ অনুসরণ করি:

প্রথমে document.createElement() দিয়ে এলিমেন্টটি তৈরি করি।

তারপর সেটির ভেতরে কোনো লেখা বা ডিজাইন দেওয়ার জন্য innerText বা classList ব্যবহার করি।

সবশেষে appendChild() বা append() ব্যবহার করে সেটিকে আমাদের পেজের নির্দিষ্ট কোনো জায়গায় (Parent element) বসিয়ে দেই।

৩. Event Bubbling কি? (সংশোধন)
Event Bubbling হলো ইভেন্ট প্রবাহের একটি প্রক্রিয়া। যখন আমরা কোনো ছোট এলিমেন্টে (যেমন বাটন) ক্লিক করি, সেই ক্লিক ইভেন্টটি শুধু ওই বাটনেই সীমাবদ্ধ থাকে না। সেটি ধীরে ধীরে উপরের দিকে উঠতে থাকে এবং তার Parent, Grandparent এমনকি body বা window পর্যন্ত পৌঁছে যায়। একেই বাবলিং বলে।

৪. Event Delegation কেন গুরুত্বপূর্ণ?
অনেক সময় আমাদের পেজে অনেকগুলো বাটন বা আইটেম থাকে। সবার জন্য আলাদা আলাদা addEventListener না লিখে আমরা যদি তাদের সবার Parent এলিমেন্টে একটি মাত্র লিসেনার বসাই, তবে তাকেই Event Delegation বলে।

সুবিধা:

কোড অনেক কম লিখতে হয়।

ব্রাউজারের মেমোরি কম খরচ হয় (Performance ভালো থাকে)।

নতুন কোনো এলিমেন্ট পরে যোগ করলেও সেটি অটোমেটিক কাজ করে।

৫. preventDefault() এবং stopPropagation() এর পার্থক্য:
preventDefault(): এটি কোনো এলিমেন্টের জন্মগত বা ডিফল্ট কাজ থামিয়ে দেয়। যেমন: লিঙ্কে ক্লিক করলে যাতে অন্য পেজে না যায়।

stopPropagation(): এটি ইভেন্টকে উপরের দিকে (Parent-এর কাছে) যাওয়া বন্ধ করে দেয়। অর্থাৎ, এটি ইভেন্ট বাবলিং (Event Bubbling) থামিয়ে দেয়।
